# SFIA Fast Track — Notion Target Content Map

## 1. Objectif

Ce document prépare la future mise à jour de Notion comme IDE métier de la SFA.

Notion ne doit pas être un export brut du repo.

Il doit devenir une base éditorialisée, claire et exploitable par :

- Morris ;
- les futurs contributeurs ;
- la SFA comme moteur d'intelligence ;
- les futurs workflows d'orchestration.

## 2. Structure cible Notion

```text
SFA / SFIA Fast Track
├── 01 — Vision générale
├── 02 — Méthode de livraison
├── 03 — Règles & garde-fous
├── 04 — Architecture & standards
├── 05 — UI & Figma Design-first
├── 06 — Automation Architecture
├── 07 — Templates & checklists
├── 08 — Cas de référence Interv360
├── 09 — Documentation plateforme
├── 10 — Roadmap d'amélioration
└── 11 — Archives / historique
```

## 4. Exclusions

Ne pas publier brut :

- logs Notion ;
- anciens `inc-*` ;
- stubs archivés ;
- documents historiques non consolidés ;
- inventaires bruts ;
- fichiers techniques ;
- statuts de merge détaillés.

## 5. Décision

Notion doit devenir l'IDE métier de la SFA sur la base du workspace nettoyé, et non un miroir complet du repo.

Référence complémentaire : `notion-publication-plan.md`

## 6. Méthodes et pratiques

La base Notion doit distinguer :

- la méthode active SFIA Fast Track ;
- les méthodes complémentaires ;
- les pratiques plateforme par rôle ou discipline.

Sources :

- `method/sfia-fast-track/`
- `method/complementary/`
- `docs/practices/`
